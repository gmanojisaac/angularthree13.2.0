# Welcome Srinivasan for the discussion on Typescript doubts

Angular-Three is a collection of packages that provide different THREE.js functionalities or 
we can state it as the three.js integration for Angular

The renderer in Angularthree examples is usually webGL. But the it was based on OPENGL which was developed for 1990s era GPU.
Latest 2015's GPUs added a shared Memory so that a new API(WEBGPU) was developed by Google to support it.

I wanted to run the Angular-Three examples in WEBGPU which was stated to run 6.2X faster than using existing webgl renderer API.

I raised an enhancement issue with Angular-Three team and it was accepted and they asked me to develop some examples of Three.js + webGPU.
Please check my issue: https://github.com/nartc/angular-three/issues/121

The First step needed to meet my objective is to understand existing TCs (in Threejs + Angular-Three + webGL). Here I am facing problems in understanding their highly reusable TS code.

So My main Challenge is to understand the State Management in Angular-Three examples for Webgl:

They have used a combination of rx-Angular and ngRx. They also use Angular-Storybook to showcase their webgl examples.

But Angular-Storybook is written using Ngxs State Management Tool which is a higher level typescript implementation of NgRx.
